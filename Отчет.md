# Отчет по практической работе

## Разработка CLI-утилиты MovieTracker CLI с использованием ИИ-инструментов

**Проект:** MovieTracker CLI  
**Язык разработки:** Python  
**Формат приложения:** CLI, Command Line Interface  
**Сравниваемые версии:** Claude Edition (Sonnet 4.6) и Codex Edition (GPT 5.5)

---

## 1. Введение

Целью практической работы была разработка CLI-утилиты на Python с использованием двух разных ИИ-инструментов и последующее сравнение полученных результатов.

В качестве темы был выбран проект **MovieTracker CLI** — консольная утилита для поиска, отслеживания и анализа кинофильмов и сериалов. Приложение позволяет искать контент, вести персональный список просмотра, выставлять оценки и получать рекомендации.

Техническое задание изначально ориентировалось на API **The Movie Database (TMDB)**. По итогам работы обе модели адаптировали реализацию под **Kinopoisk API Unofficial** — отечественный аналог с аналогичной структурой, но без OAuth-сессии пользователя и облачного watchlist. Это решение обеих моделей стало первым заметным расхождением с исходным ТЗ, но устранило необходимомть в vpn или прокси.

Дополнительно работа позволила сравнить не только итоговый код, но и сам процесс разработки с помощью ИИ: глубину покрытия ТЗ, архитектурные решения, тестовое покрытие и выбор инструментов.

---

## 2. Цель и задачи работы

### Цель

Разработать две независимые версии CLI-приложения для отслеживания фильмов и сериалов, сравнить подходы к реализации и оценить эффективность использованных ИИ-инструментов.

### Основные задачи

- составить техническое задание для CLI-утилиты;
- реализовать две версии проекта независимо друг от друга;
- использовать внешний REST API для получения данных о фильмах;
- реализовать локальное хранение watchlist, оценок и заметок;
- добавить кэширование запросов к API;
- реализовать обработку ошибок;
- подготовить тесты;
- сравнить две реализации по техническим и процессным критериям;
- сделать вывод о том, какой инструмент оказался эффективнее.

---

## 3. Техническое задание

Согласно ТЗ, приложение **MovieTracker CLI** должно быть консольной программой на Python, которая позволяет искать, отслеживать и анализировать фильмы и сериалы через внешний REST API.

### Обязательные команды CLI

| Команда | Назначение |
|---|---|
| `search` | Поиск фильмов и сериалов по названию |
| `details` | Подробная информация: описание, актёры, рецензии, похожие |
| `add-watchlist` | Добавить фильм в список просмотра |
| `list-watchlist` | Просмотреть список с фильтрацией и статистикой |
| `rate` | Выставить оценку и добавить отзыв |
| `recommend` | Получить рекомендации по фильму или жанру |
| `popular` | Топ популярных фильмов и сериалов |
| `trending` | Трендовый контент за день или неделю |
| `auth` | Управление API-ключом |
| `cache` | Управление локальным кэшем |

### Требования к данным

- Внешний REST API для поиска и получения информации о фильмах
- Локальное хранилище watchlist в JSON-формате
- Кэширование ответов API с дифференцированным TTL
- Форматы вывода: таблица, JSON, CSV

### Нефункциональные требования

- Python 3.11+;
- читаемый вывод в консоли (Rich);
- соответствие PEP 8;
- обработка сетевых ошибок и ошибок API;
- покрытие кода тестами не менее 80%;
- совместимость с macOS, Linux и Windows.

## 4. Методология сравнения

Разработка велась в двух отдельных папках:

- `Claude/` — версия, полученная при работе с Claude Sonnet 4.6;
- `Codex/` — альтернативная версия, полученная при работе с GPT 5.5.

Обе версии решали одну и ту же задачу и ориентировались на одно техническое задание. При сравнении учитывались следующие критерии:

- полнота выполнения ТЗ;
- реализованные команды CLI;
- архитектура и качество кода;
- количество и структура тестов;
- выбор инструментов сборки;
- наличие CI/CD и инфраструктуры;
- удобство запуска и документации.

## 5. Реализация Claude Edition

Версия `Claude` представляет собой CLI-приложение с акцентом на строгую типизацию и асинхронные HTTP-запросы.

### Используемые источники данных

- **Kinopoisk API Unofficial** — поиск фильмов, детали, похожие, популярное, тренды;
- **Локальное хранилище** — watchlist, оценки, заметки, теги (TinyDB + JSON).

### Технологический стек

| Компонент | Технология | Версия |
|---|---|---|
| CLI-фреймворк | Typer | 0.12+ |
| HTTP-клиент | httpx (async) + anyio | 0.27+ |
| Вывод в терминал | Rich | 13+ |
| Кэширование | diskcache | 5.6+ |
| Локальная БД | TinyDB | 4.8+ |
| Конфигурация | dynaconf | 3.2+ |
| Логирование | loguru | 0.7+ |
| Сборка | setuptools | 68+ |

### Основные файлы проекта

| Файл / Модуль | Назначение |
|---|---|
| `src/movie_tracker/main.py` | Точка входа, Typer app |
| `src/movie_tracker/cli/` | Команды: search, details, watchlist, auth, actions |
| `src/movie_tracker/api/` | HTTP-клиент Kinopoisk: search, details, auth |
| `src/movie_tracker/storage/` | TinyDB watchlist + хранение токена |
| `src/movie_tracker/cache/` | diskcache менеджер |
| `src/movie_tracker/formatters/` | Rich-таблицы |
| `src/movie_tracker/models/` | dataclass-модели |

### Особенности реализации

Версия Claude выделяется строгой слоистой архитектурой: слой CLI содержит только UI-логику, API-слой отвечает за HTTP и маппинг ответов, хранилище работает независимо. Каждая команда запускает `asyncio.run()` — это делает Typer-приложение синхронным снаружи, но полностью async внутри.

HTTP-клиент строит URL явно — `https://kinopoiskapiunofficial.tech/api/v2.2/films` — без base_url, что исключает неочевидное поведение httpx при путях со слешем.

Кэширование реализовано с дифференцированным TTL: поиск — 60 минут, детали фильма — 24 часа, популярное — 3 часа, тренды — 30 минут, рекомендации — 12 часов.

### Тесты

В версии Claude 85 тестов. Они покрывают:

- модели данных (SearchResult, WatchlistEntry и другие);
- валидаторы (оценки, теги, год, страница, приоритет);
- операции хранилища (TinyDB CRUD);
- все CLI-команды через CliRunner.

### Отклонение от ТЗ

ТЗ предполагало TMDB с OAuth-аутентификацией и облачным watchlist. Claude адаптировал реализацию под Kinopoisk API Unofficial, заменив OAuth на API-ключ, а облачный watchlist — на локальный TinyDB.

### Итог по Claude Edition

Версия Claude отличается высоким тестовым покрытием и чёткой архитектурой. Async-подход усложняет точку входа, но обеспечивает эффективную работу с сетью. Код хорошо разделён по слоям и легко поддаётся сопровождению.

## 6. Реализация Codex Edition

Версия `Codex` реализует CLI-приложение с упором на полноту набора команд и готовность к распространению: есть PyInstaller-сборка и GitHub Actions CI/CD.

### Используемые источники данных

- **Kinopoisk API Unofficial** — поиск, детали, популярное, тренды, рекомендации;
- **Локальное хранилище** — watchlist, оценки, отзывы, теги (TinyDB + JSON).

### Технологический стек

| Компонент | Технология | Версия |
|---|---|---|
| CLI-фреймворк | Typer | 0.12+ |
| HTTP-клиент | httpx (sync) | 0.27+ |
| Вывод в терминал | Rich | 13+ |
| Кэширование | diskcache | 5.6+ |
| Локальная БД | TinyDB | 4.8+ |
| Конфигурация | dynaconf | 3.2+ |
| Логирование | loguru | 0.7+ |
| Сборка | hatchling | 1.25+ |
| Бинарная сборка | PyInstaller | 6.6+ |

### Основные файлы проекта

| Файл / Модуль | Назначение |
|---|---|
| `src/movie_tracker/main.py` | Точка входа, Typer app |
| `src/movie_tracker/cli/` | 10 модулей: search, details, watchlist, auth, popular, trending, recommend, cache, errors, state |
| `src/movie_tracker/api/` | 8 модулей: search, details, auth, popular, trending, recommendations, watchlist, client |
| `src/movie_tracker/storage/` | TinyDB watchlist + хранение токена |
| `src/movie_tracker/cache/` | diskcache менеджер |
| `src/movie_tracker/formatters/` | Rich-таблицы, JSON, CSV |
| `packaging/movie-tracker.spec` | PyInstaller spec для бинарной сборки |
| `.github/workflows/` | CI (lint, tests) и release (PyInstaller бинарь) |

### Особенности реализации

Версия Codex построена с более детальным разделением: каждая команда и каждый API-эндпоинт вынесены в отдельный модуль. HTTP-запросы выполняются синхронно, что упрощает структуру кода.

Отдельного внимания заслуживает инфраструктура: GitHub Actions автоматически запускает линтер и тесты при push, а при создании тега вида `v1.0.0` собирает PyInstaller-бинарь для Windows, Linux и macOS. Это позволяет распространять приложение без необходимости устанавливать Python.

Конфигурация API-ключа поддерживает три переменные окружения: `MOVIE_TRACKER_API__KEY`, `KINOPOISK_API_KEY` и `KINOPOISK_UNOFFICIAL_API_KEY`, что делает интеграцию в разные среды удобнее.

### Тесты

В версии Codex 12 тестов. Они покрывают:

- операции watchlist (добавление, список, оценка, статистика);
- поиск через API (нормализация ответа, приоритет токенов);
- CLI-команды (version, init, auth token, list-watchlist);
- валидаторы (оценки, статус, тип, приоритет).

### Отклонение от ТЗ

Аналогично Claude, Codex заменил TMDB на Kinopoisk API Unofficial. Дополнительно: команда `recommend` использует похожие фильмы из Kinopoisk или локальные жанровые предпочтения из watchlist вместо TMDB Recommendations API.

### Итог по Codex Edition

Версия Codex выделяется более полным разделением кода по файлам и готовой инфраструктурой для распространения. PyInstaller-сборка и GitHub Actions делают проект удобным для конечных пользователей. Количество тестов значительно ниже, чем у Claude Edition.

## 8. Проверка итогового результата

Финальный сценарий работы был проверен для обеих версий.

### Claude Edition

```bash
cd Claude
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
movie-tracker-claude init
movie-tracker-claude auth token ВАШ_КЛЮЧ
movie-tracker-claude search "Интерстеллар"
movie-tracker-claude details 258687
movie-tracker-claude add-watchlist 258687 --priority 5 --tags "фантастика,нолан"
movie-tracker-claude list-watchlist --stats
movie-tracker-claude rate 258687 9.5 --review "Лучший фильм Нолана"
movie-tracker-claude recommend --based-on 258687 --limit 5
```

### Codex Edition

```bash
cd Codex
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
movie-tracker init
movie-tracker auth token ВАШ_КЛЮЧ
movie-tracker search "Интерстеллар"
movie-tracker details 258687 --section all
movie-tracker add-watchlist 258687 --priority 5 --tags "фантастика,нолан"
movie-tracker list-watchlist --stats
movie-tracker rate 258687 9.5 --review "Лучший фильм Нолана"
movie-tracker recommend --based-on 258687 --limit 5
```

Обе версии выполняют основной сценарий: поиск, просмотр деталей, добавление в watchlist, выставление оценки и получение рекомендаций. Данные хранятся локально в `~/.movie-tracker/`.

## 9. Особенности сравнения

Обе версии приняли одно и то же ключевое решение — заменить TMDB на Kinopoisk API Unofficial. Это привело к схожим ограничениям: нет OAuth пользователя, нет облачного watchlist, рекомендации основаны на локальных предпочтениях или похожих фильмах Kinopoisk.

Claude Sonnet 4.6 сделал ставку на качество: строгую типизацию, async-архитектуру и максимальное тестовое покрытие (85 тестов). Это делает код надёжным и предсказуемым, но требует больше усилий при первом знакомстве с проектом.

GPT 5.5 сделал ставку на полноту и инфраструктуру: больше раздельных CLI-модулей, PyInstaller-сборка и готовый GitHub Actions pipeline. Для конечного пользователя Codex Edition удобнее в распространении, но тестовое покрытие значительно ниже.

Ключевое различие не в наборе команд (он почти идентичен), а в приоритетах: Claude — надёжность и тестируемость, Codex — полнота и готовность к деплою.

## 10. Итоговый вывод

Обе версии проекта успешно реализуют требуемую CLI-утилиту. Они соответствуют основной постановке задачи: используют внешний REST API, реализуют полный набор команд, хранят данные локально и корректно обрабатывают ошибки.

**Claude Sonnet 4.6** создал более надёжную с точки зрения тестирования версию: 85 тестов против 12, строгая типизация, чёткое разделение на слои. Async-архитектура добавляет сложности, но обеспечивает эффективную работу с сетью. Эта версия лучше подходит для долгосрочной поддержки и развития.

**GPT 5.5** создал более готовую к распространению версию: PyInstaller-бинарь, GitHub Actions CI/CD, поддержка CSV-формата вывода. Разбивка по отдельным модулям делает каждый файл компактным. Эта версия лучше подходит для быстрого релиза и доступности для конечных пользователей без Python.

Главный вывод: **Claude** оказался сильнее по качеству кода и тестируемости. **GPT** оказался сильнее по инфраструктуре и готовности к распространению. Выбор зависит от цели: если важна надёжность — Claude, если важна доступность — GPT.
